## 测试执行指南

### 运行单元测试
```bash
# 运行所有测试
npm test

# 运行特定测试文件
npm test -- src/__tests__/game/board.test.ts

# 运行测试并生成覆盖率报告
npm run test:coverage

# 运行测试UI
npm run test:ui
```

### 运行E2E测试
```bash
# 运行所有E2E测试
npm run test:e2e

# 运行特定E2E测试
npx playwright test pvp-game.spec.ts

# 查看E2E测试报告
npx playwright show-report
```

## 测试最佳实践

### 1. TDD流程
- ✅ 先写测试，看到失败（红）
- ✅ 实现功能或配置，让测试通过（绿）
- ✅ 重构优化
- ✅ 确保所有测试保持通过

### 2. 测试命名
- 使用描述性的测试名称
- 格式：`应该[期望的行为]当[条件]`
- 示例：`应该返回错误当坐标超出边界`

### 3. 测试结构
- 遵循AAA模式（Arrange-Act-Assert）
- 每个测试只验证一个行为
- 保持测试简短和专注
