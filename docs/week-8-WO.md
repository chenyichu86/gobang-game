## 2. 工作对象分解

### 2.1 子工作对象1：金币系统

**目标**: 实现金币获取和管理

**交付物**:
- `src/services/coin-service.ts` - 金币服务
- `src/types/economy.ts` - 经济系统类型定义
- `src/store/user-store.ts` (扩展) - 添加金币字段

**技术规格**:
- **金币获取规则**:
  - 胜利奖励：+10金币
  - 失败奖励：+2金币
  - 和棋奖励：+5金币
  - 每日签到：+50金币
  - 成就奖励：+50~500金币（根据成就难度）
- **金币上限**：无上限
- **金币用途**：
  - 购买皮肤（Week 8）
  - 购买道具（Week 9+）

**验收标准**:
- [ ] 金币计算正确（胜/负/和棋）
- [ ] 每日签到奖励正确
- [ ] 成就奖励正确发放
- [ ] 金币数据正确保存和加载
- [ ] 向后兼容Week 1-7的数据

---

### 2.2 子工作对象2：每日任务系统

**目标**: 实现每日任务和周常任务

**交付物**:
- `src/services/task-service.ts` - 任务服务
- `src/types/task.ts` - 任务类型定义
- `src/constants/tasks.ts` - 任务配置

**技术规格**:
- **每日任务**（3个）:
  1. 完成游戏达人 - 完成3局游戏（30金币）
  2. 胜利者 - 获得1局胜利（50金币）
  3. 博学先锋 - 使用1次提示（20金币）
- **周常任务**（1个）:
  - 千胜之路 - 累计获胜10局（200金币 + 限定皮肤）
- **任务刷新**:
  - 每日任务：每天0点刷新
  - 周常任务：每周一0点刷新
- **任务进度追踪**:
  - 实时更新任务进度
  - 完成后可领取奖励
  - 已领取的任务标记为已完成

**验收标准**:
- [ ] 每日任务正确生成和刷新
- [ ] 周常任务正确生成和刷新
- [ ] 任务进度正确追踪
- [ ] 任务奖励正确发放
- [ ] 任务数据正确保存
- [ ] 向后兼容Week 1-7的数据

### 2.3 子工作对象3：商城系统

**目标**: 实现商城界面和皮肤购买

**交付物**:
- `src/store/shop-store.ts` - 商城状态管理
- `src/components/Shop/` - 商城UI组件
  - `ShopPage.tsx` - 商城页面
  - `SkinCard.tsx` - 皮肤卡片
  - `CoinDisplay.tsx` - 金币显示
- `src/types/shop.ts` - 商城类型定义
- `src/constants/skins.ts` - 皮肤配置

**技术规格**:
- **商品类型**:
  - 棋盘皮肤（5款）
  - 棋子皮肤（5款）
- **皮肤价格**:
  - 普通皮肤：100-500金币
  - 稀有皮肤：1000-2000金币
  - 限定皮肤：通过任务获得
- **购买逻辑**:
  - 检查金币余额
  - 扣除金币
  - 解锁皮肤
  - 保存数据
- **默认皮肤**：
  - 经典木纹棋盘（已拥有）
  - 黑白棋子（已拥有）

**验收标准**:
- [ ] 商城界面正常显示
- [ ] 皮肤列表正确渲染
- [ ] 购买逻辑正确
- [ ] 金币余额不足时禁用购买
- [ ] 已拥有皮肤标记为"已拥有"
- [ ] 皮肤正确应用到游戏中
- [ ] 向后兼容Week 1-7的功能

### 2.4 子工作对象4：每日签到功能

**目标**: 实现每日签到和连续签到奖励

**交付物**:
- `src/services/checkin-service.ts` - 签到服务
- `src/components/CheckIn/` - 签到UI组件
  - `CheckInButton.tsx` - 签到按钮
  - `CheckInCalendar.tsx` - 签到日历

**技术规格**:
- **签到奖励**:
  - 每日签到：+50金币
  - 连续7天：额外+100金币（成就"坚持不懈"）
  - 连续30天：额外+500金币 + 限定皮肤
- **签到规则**:
  - 每天只能签到1次
  - 0点后可重新签到
  - 签到后获得金币
- **连续签到中断**:
  - 中断后重新开始计算
  - 提示用户保持连续签到

**验收标准**:
- [ ] 签到按钮正常工作
- [ ] 每日只能签到1次
- [ ] 连续签到天数正确计算
- [ ] 签到奖励正确发放
- [ ] 签到日历正确显示
- [ ] 向后兼容Week 1-7的数据

## 3. 技术约束

### 3.1 存储约束
- LocalStorage容量限制：5-10MB
- 需要处理存储空间不足的情况
- 数据版本控制（支持未来迁移）

### 3.2 性能约束
- 金币计算响应时间：<50ms
- 任务刷新响应时间：<100ms
- 商城购买响应时间：<100ms
- 不影响游戏性能（AI响应时间）

### 3.3 兼容性约束
- 不能破坏Week 1-7的任何功能
- 向后兼容现有用户数据
- 新功能可选启用（不影响核心玩法）
