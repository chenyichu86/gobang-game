### 2.1 子工作对象1：数据持久化

**目标**: 实现LocalStorage持久化，数据不丢失

**交付物**:
- `src/services/storage-service.ts` - 存储服务扩展
- `src/types/storage.ts` - 存储类型定义
- `src/store/user-store.ts` (扩展) - 添加持久化逻辑

**技术规格**:
- **LocalStorage键值设计**:
  ```typescript
  const STORAGE_KEYS = {
    USER_DATA: 'gobang_user_data_v2',  // 金币、等级、成就
    TASKS: 'gobang_tasks_v2',           // 任务数据
    CHECK_IN: 'gobang_check_in_v2',      // 签到数据
    SHOP: 'gobang_shop_v2',             // 商城数据（皮肤拥有状态）
  };
  ```

- **数据自动保存**:
  - 金币变化时自动保存
  - 任务进度更新时自动保存
  - 签到后自动保存
  - 购买皮肤后自动保存

- **数据自动加载**:
  - 应用启动时自动加载
  - 数据版本控制（v2）
  - 数据迁移支持（v1 → v2）

- **错误处理**:
  - LocalStorage不可用时的降级方案
  - 数据损坏时的恢复机制
  - 存储空间不足时的处理

**验收标准**:
- [ ] 数据正确保存到LocalStorage
- [ ] 刷新页面数据不丢失
- [ ] 数据版本控制正常工作
- [ ] 数据迁移正确（从Week 7/8数据）
- [ ] 错误处理完善

**目标**: 集成到游戏流程，游戏结束触发金币和任务

- `src/store/game-store.ts` (扩展) - 添加金币和任务触发
- `src/store/user-store.ts` (扩展) - 添加持久化监听

**技术规格**:

**游戏结束流程扩展**:
```typescript
// Week 7流程
游戏结束 → 判断胜负 → 计算经验值 → 检查等级升级 → 检查成就 → 显示结果 → 保存数据

// Week 9流程
游戏结束 → 判断胜负 → 计算金币 → 更新任务进度 → 检查成就 → 计算经验值 → 检查等级升级 → 显示结果 → 保存数据
```

**金币奖励触发**:
```typescript
// game-store.ts
endGame(result: GameResult): void {
  // 1. 计算金币
  const coins = coinService.calculateCoinGain(result);
  userStore.addCoins(coins);

  // 2. 更新任务进度
  userStore.checkTaskProgress(result); // 'win' | 'lose' | 'draw'

  // 3. 原有的经验值和成就逻辑
  const exp = expService.calculateExpGain(result);
  userStore.addExp(exp);

  // ...
}
```

**数据自动保存**:
```typescript
// user-store.ts
addCoins(amount: number): void {
  this.coins += amount;
  this.totalEarned += amount;

  // 自动保存到LocalStorage
  storageService.saveUserData(this.getUserData());
}
```
