# 网页端五子棋游戏 - 产品需求文档（PRD）

## 一、项目概述

### 1.1 项目背景
开发一个基于Web的五子棋游戏，提供人机对战和双人对战功能，旨在为用户提供轻松愉快的棋类游戏体验。

### 1.2 目标用户
- 休闲游戏玩家
- 棋类游戏爱好者
- 寻找轻松娱乐的网页用户

### 1.3 核心价值
- 随时随地进行五子棋对弈
- 提供多难度AI对手
- 流畅的用户体验和精美的界面设计

## 二、游戏规则设计

### 2.1 基本规则
- **棋盘规格**：标准15×15交叉点棋盘
- **获胜条件**：任意方向（横、竖、斜）连成5个或以上同色棋子即获胜
- **先手规则**：黑棋先手，白棋后手
- **落子规则**：
  - 双方轮流落子
  - 棋子落在交叉点上
  - 落子后不可移动或移除

### 2.2 特殊规则（可选）
- **禁手规则**（高级模式）：
  - 黑棋禁止"三三"（同时形成两个活三）
  - 黑棋禁止"四四"（同时形成两个四）
  - 黑棋禁止"长连"（连成6个或以上）
- **和棋规则**：双方均无法获胜时，判定为和棋

### 2.3 游戏模式
1. **人机对战**
   - 玩家执黑先行
   - AI执白后行
2. **双人对战**
   - 本地双人同屏对战
   - 在线对战（后期扩展）

## 三、用户交互设计

### 3.1 游戏流程
```
开始页面 → 选择模式 → 选择难度 → 游戏进行中 → 结算页面 → 重新开始/返回主菜单
```

### 3.2 核心交互

#### 3.2.1 开始页面
- 标题和游戏Logo
- 三个主要按钮：
  - [人机对战]
  - [双人对战]
  - [游戏设置]
- 底部显示：历史战绩、当前等级

#### 3.2.2 模式选择
**人机对战：**
- 难度选择：简单/中等/困难/大师
- 执棋选择：先手（黑棋）/后手（白棋）
- 显示当前难度胜率统计

**双人对战：**
- 玩家1昵称输入
- 玩家2昵称输入
- 先手选择

#### 3.2.3 游戏界面交互
- **落子**：点击棋盘交叉点落子
- **悔棋**：每方限用3次悔棋机会
- **提示**：AI推荐落子位置（每日3次免费）
- **认输**：直接结束游戏
- **暂停**：暂停游戏计时（仅限双人对战）
- **音效开关**：实时切换

#### 3.2.4 结算页面
- 显示获胜方
- 本局用时统计
- 棋局回放功能
- 保存棋谱按钮
- [再来一局] [返回主页] 按钮

### 3.3 操作反馈
- **落子反馈**：
  - 音效：落子声
  - 视觉：最新落子高亮显示
  - 动画：棋子落下动画（缩放+阴影）
- **错误提示**：
  - 无效位置提示（已有棋子）
  - 悔棋次数用尽提示
  - 震动反馈（移动端）
- **胜利特效**：
  - 连成五子的棋子闪烁
  - 烟花粒子效果
  - 胜利音效

## 四、游戏界面设计

### 4.1 设计风格
- **整体风格**：简洁现代，木质纹理与扁平化结合
- **配色方案**：
  - 主色：温暖木色（#D2B48C, #DEB887）
  - 棋盘：浅米色（#F5DEB3）
  - 黑棋：深灰（#2C2C2C）带高光
  - 白棋：纯白（#FFFFFF）带阴影
  - 强调色：金色（#FFD700）用于胜利特效

### 4.2 布局设计

#### 4.2.1 主界面布局
```
┌────────────────────────────────────┐
│            五子棋                   │
│        [Logo/图标]                  │
│                                    │
│      [人机对战]  [双人对战]          │
│                                    │
│      [游戏设置]  [排行榜]            │
├────────────────────────────────────┤
│  等级: ⭐⭐⭐ | 胜率: 65%           │
└────────────────────────────────────┘
```

#### 4.2.2 游戏界面布局
```
┌────────────────────────────────────┐
│  ⏱️ 黑方: 05:23  |  白方: 04:15 ⏱️  │
├────────────────────────────────────┤
│                                    │
│                                    │
│         15×15 棋盘区域              │
│      (居中显示，响应式缩放)          │
│                                    │
│                                    │
├────────────────────────────────────┤
│  [悔棋] [提示] [暂停] [认输] [设置]  │
└────────────────────────────────────┘
```

### 4.3 UI组件设计

#### 4.3.1 棋盘设计
- **背景**：木质纹理，带网格线
- **星位点**：5个黑点标记（天元+四角星位）
- **网格线**：深棕色细线，线宽1-2px
- **坐标标识**：边缘显示A-O、1-15坐标（可选显示）
- **响应式**：自适应屏幕大小，保持正方形比例

#### 4.3.2 棋子设计
- **黑棋**：
  - 径向渐变（深灰到黑色）
  - 顶部高光（30%透明度白色）
  - 投影效果
  - 直径约为格子间距的80%
- **白棋**：
  - 纯白色背景
  - 柔和阴影
  - 边缘描边（浅灰）

#### 4.3.3 按钮组件
- **主按钮**（人机对战、双人对战）：
  - 圆角矩形，8px圆角
  - 木纹背景 + 金色边框
  - Hover: 轻微放大 + 阴影增强
- **次要按钮**（悔棋、提示）：
  - 扁平化设计
  - 图标 + 文字
  - 禁用状态: 降低透明度

#### 4.3.4 信息面板
- **计时器**：数字时钟格式，mm:ss
- **当前回合指示器**：高亮当前行动方
- **难度指示器**：星星图标显示当前难度

### 4.4 响应式设计
- **桌面端（>768px）**：棋盘最大600×600px
- **平板端（768px-1024px）**：棋盘占屏幕宽度80%
- **移动端（<768px）**：棋盘占屏幕宽度95%，按钮排列改为竖向

### 4.5 动画设计
- **页面切换**：淡入淡出（300ms）
- **棋子落下**：从上方掉落（scale: 1.5 → 1.0, 200ms）
- **胜利连线**：棋子依次发光（每个100ms延迟）
- **按钮Hover**：scale 1.05（150ms ease-out）

## 五、奖励系统设计

### 5.1 经验值系统
- **获胜**：+100 经验值
- **失败**：+20 经验值
- **和棋**：+50 经验值
- **连胜奖励**：连胜额外+50 经验值/局

### 5.2 等级系统
| 等级 | 称号 | 所需经验值 | 解锁内容 |
|------|------|------------|----------|
| 1 | 初学者 | 0 | 基础功能 |
| 2 | 入门棋手 | 500 | 简单AI |
| 3 | 进阶棋手 | 1500 | 中等AI |
| 4 | 高级棋手 | 3000 | 困难AI |
| 5 | 棋艺大师 | 6000 | 大师AI |
| 6 | 宗师 | 12000 | 特殊棋盘皮肤 |

### 5.3 成就系统
**对局成就：**
- 🏆 初出茅庐：完成1局游戏
- 🎯 首胜：获得首次胜利
- 🔥 连胜大师：达成5连胜
- 💪 坚韧不拔：连续完成10局游戏

**技巧成就：**
- ⚡ 闪电战：60秒内获胜
- 🛡️ 完美防守：在劣势下翻盘
- 🎨 艺术家：用5步以内获胜（与弱AI）

**收集成就：**
- 📚 博学者：完成100局游戏
- 👑 千胜王者：累计获胜1000局

### 5.4 虚拟货币（金币）
- **获取方式**：
  - 每局胜利：+10 金币
  - 每日签到：+50 金币
  - 完成成就：+100-500 金币
  - 观看广告（可选）：+20 金币/次

- **用途**：
  - 购买提示次数：10 金币/次
  - 购买悔棋机会：20 金币/次
  - 解锁棋盘皮肤：200-1000 金币
  - 解锁棋子皮肤：100-500 金币

### 5.5 每日任务
- **任务1**：完成3局游戏 - 奖励：30金币
- **任务2**：获得1局胜利 - 奖励：50金币

- **周常任务**：累计获胜10局 - 奖励：200金币 + 稀有皮肤

## 六、难度等级设计

### 6.1 难度分级

#### 简单难度（⭐）
- **目标用户**：完全新手
- **AI策略**：
  - 80%概率随机落子
  - 20%概率进行基本防守（阻挡玩家3连）
  - 不会主动进攻
- **预期胜率**：90%+

#### 中等难度（⭐⭐）
- **目标用户**：了解基本规则的玩家
- **AI策略**：
  - 识别并阻挡玩家的3连和4连
  - 尝试形成自己的2连和3连
  - 优先防守，偶尔进攻
- **预期胜率**：60-70%

#### 困难难度（⭐⭐⭐）
- **目标用户**：有一定经验的玩家
- **AI策略**：
  - 攻守兼备
  - 使用评分系统评估每个位置
  - 考虑多步后的局面（2步前瞻）
  - 识别双三、四三等棋型
- **预期胜率**：30-40%

#### 大师难度（⭐⭐⭐⭐）
- **目标用户**：高水平玩家
- **AI策略**：
  - 使用Minimax算法+Alpha-Beta剪枝
  - 搜索深度4-6层
  - 高级位置评估函数
  - 考虑禁手规则
  - 学习玩家常用招式
- **预期胜率**：5-10%

### 6.2 难度选择引导
- 首次进入游戏显示难度说明
- 显示当前难度的历史胜率
- 连胜3次后提示"是否提升难度？"
- 连败3次后提示"是否降低难度？"

## 七、技术实现要点

### 7.1 前端技术栈建议
- **框架**：React / Vue.js
- **UI组件库**：Ant Design / Material-UI
- **状态管理**：Redux / Vuex
- **Canvas库**：Konva.js / Pixi.js（用于棋盘渲染）

### 7.2 AI实现方案
- **简单/中等难度**：基于规则的AI
- **困难/大师难度**：Minimax + Alpha-Beta剪枝
- **优化**：Web Worker避免阻塞UI线程

### 7.3 数据存储
- **本地存储**：LocalStorage保存用户进度、历史记录
- **云存储**（可选）：Firebase用于在线对战和排行榜

### 7.4 性能优化
- 棋盘使用Canvas渲染而非DOM节点
- AI计算使用Web Worker
- 棋谱仅记录落子坐标序列
- 实现虚拟滚动（棋局回放功能）

## 十一、风险和挑战

### 11.1 技术风险
- **AI性能**：大师难度AI可能计算耗时
  - 缓解方案：使用Web Worker，限制搜索深度
- **浏览器兼容性**：不同浏览器Canvas性能差异
  - 缓解方案：充分测试主流浏览器

### 11.2 产品风险
- **用户粘性**：单机游戏可能缺乏长期吸引力
  - 缓解方案：增加成就、收集要素，准备在线对战
- **难度平衡**：AI难度可能不合理
  - 缓解方案：大量测试，收集数据后调整
