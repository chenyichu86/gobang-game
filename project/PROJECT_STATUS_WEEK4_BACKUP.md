## 🔄 工作流程

### 标准开发流程（每个任务/功能）

```
1. 产品经理
   ├─ 编写WO文档（工作对象定义）
   ├─ 编写PL文档（产品逻辑规范）
   └─ 定义门禁标准和验收条件

2. 测试
   ├─ 根据WO/PL设计测试用例
   ├─ 先编写测试代码（TDD）
   └─ 更新事实源（测试用例清单）

3. 开发
   ├─ 读取WO/PL和测试用例
   ├─ 编写业务逻辑代码
   ├─ 运行测试，修复到全绿
   └─ 更新事实源（开发进度）

4. 测试
   ├─ 验证所有测试通过
   ├─ 生成测试报告
   └─ 更新事实源（测试结果）

5. 产品经理
   ├─ 功能验收
   ├─ 确认门禁标准达标
   └─ 决定是否冻结功能

6. 项目经理
   ├─ 主持评审会议
   ├─ 确认可以进入下一阶段
   └─ 更新事实源（阶段完成）
```
